    构造函数构建对象

    # 构造函数构建对象

    // 使用构造函数foo，new了一个实例对象
function foo() {}
const f1 = new foo();

    # prototype

    • 每个函数都有一个prototype属性，且prototype是函数才会有的属性
    • 函数的prototype属性指向了一个对象，这个对象就是调用该构造函数构建的实例对象的原型
    • 什么是原型呢？ 每一个js对象(null除外)创建的时候都会与之关联另一个对象，这个对象就是原型
      • 每一个对象都会从原型中"继承"属性
        • 但并不是真正的继承。继承意味着复制操作，但是对象的创建默认不会复制对象的属性，相反，js只是在两个对象质之间创建对个关联，这样一个对象就可以通过委托访问另一个对象的属性和方法。因此实例对象并不是以继承的形式来访问原型的

    # proto

    • 每一个js对象（null除外），都有一个属性，叫__proto__，这个属性会指向该对象的原型 person.__proto__ === Person.prototype; // true

    # constructor

    • 每个原型都有一个constructor属性指向关联的构造函数 Person === Person.prototype.constructor; // true

    # 原型链

    • 读取实例的属性时，如果找不到，就会查找与对象关联的原型上的属性，如果还查不到，就会去查找原型的原型，直到查到顶层
    • 原型其实也是一个对象，那么对象就是可以通过Object()构造函数创建的,因此原型的__proto__最终可以指向Object.prototype
    • Object.prototype的原型是什么呢？答案是null
      • Object.prototype没有原型，表示原型链查找到Object.prototype就可以停止查找了
      • Object.prototype === null; // true

    # 为什么Object.prototype === null?

    • Object.prototype也是一个对象，但他是没有__proto__属性的 Object.prototype.__proto__ === null; // true
      • 为什么要这么设定呢？由于Object.prototype是一个对象，那么就是由Object函数创建的，按照一般规则即Object.prototype.__proto__ === Object.prototype; // true，如果一直这样使Object.prototype.__proto__属性指向它自身，原型链就陷入死循环没有终点了，
      • 为了让原型链有终点，在原型链的最顶端，JavaScript规定了Object.prototype.proto === null

    # 特殊的Function函数

    以上的情况为一般情况，一般规则

      function foo() {}
const f1 = new foo();
// 对象的__proto__指向函数原型
f1.__proto__ === foo.prototype
// 原型链，原型也是一个对象，最终可找到Object.prototype
foo.prototype.__proto__ === Object.prototype
// 原型的constructor指向函数
foo.prototype.constructor === foo
    • 众所周知，Function函数其实也是对象，作为一个对象那么就拥有__proto__属性，加上函数都是由Function函数生成的，可以得到下面的代码
          function foo(){};
    foo.__proto__ === Function.prototype; // true
      • 因此，Function()作为一个对象是适用一般规则的
          // 函数构造方法作为对象时，生成他的即是它本身
    Function.__proto__ === Function.prototype
      • Object()这个构造函数作为一个对象同样可以得出：
          // Object构造函数是由Function函数生成的
    Object.__proto__ === Function.prototype

    那么Function.prototype的指向又是什么呢？

    1. Function.prototype的指向
    • 一般函数默认的prototype是一个对象，它有两个属性：constructor和 __proto__ 。其中constructor属性指向这个函数自身，__proto__属性指向Object.prototype，这其实是因为普通函数的prototype属性即原型是一个对象，是由Object函数生成的。
    • 函数构造函数很特殊，我们可以发现typeof Function.prototype === 'function' // true 为什么他不像其他函数一样，原型是一个对象，而是一个函数呢？那究竟是怎么样的一个函数
    // 函数内部是[native code]，也就是系统编译好的二进制代码函数，这就暂时没法深究了
ƒ () { [native code] }

    来看看`Function.prototype``又指向谁呢？

    Function.prototype.__proto__ === Object.prototype // true

    由此我们可以总结，若按照一般规则Function.prototype.__proto__ === Function.prototype; // true，这个同样会使得函数原型链没有终点 - JavaScript规定Function.prototype.proto === Object.prototype，这样既避免了出现循环引用，又让__proto__构成的原型链指向了唯一的终点：Object.prototype.proto === null
